# API Тест-кейсы

## POST /api/1/item

### TC-001: Успешное создание объявления
**Шаги:**
1. Отправить POST-запрос на `/api/1/item`.
2. Тело запроса:
```json
{
  "sellerID": 555666,
  "name": "Смартфон на Android",
  "price": 25000,
  "statistics": { "likes": 5, "viewCount": 20, "contacts": 2 }
}
```
**Ожидаемый результат:** 200 OK. Поле `status` содержит UUID созданного объявления.

---

### TC-002: Создание объявления с пустым именем
**Шаги:**
1. Отправить POST-запрос на `/api/1/item`.
2. Тело запроса:
```json
{
  "sellerID": <случайный 111111–999999>,
  "name": "",
  "price": 6000,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 2 }
}
```
**Ожидаемый результат:** 400 Bad Request.

---

### TC-003: Создание объявления с отрицательной ценой
**Шаги:**
1. Отправить POST-запрос на `/api/1/item`.
2. Тело запроса:
```json
{
  "sellerID": <случайный 111111–999999>,
  "name": "Наушники",
  "price": -100,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 2 }
}
```
**Ожидаемый результат:** 400 Bad Request.

---

### TC-004: Идемпотентность POST — два одинаковых запроса создают разные объявления
**Шаги:**
1. Дважды отправить POST-запрос на `/api/1/item` с одним и тем же телом.
2. Тело запроса:
```json
{
  "sellerID": <случайный 111111–999999>,
  "name": "Книга по программированию",
  "price": 800,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 2 }
}
```
**Ожидаемый результат:** 200 OK оба раза. Каждый раз возвращается уникальный UUID.

---

### TC-005: Создание объявления с неверным типом sellerID (строка вместо числа)
**Шаги:**
1. Отправить POST-запрос на `/api/1/item`.
2. Тело запроса:
```json
{
  "sellerID": "один",
  "name": "Книга по программированию",
  "price": 800,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 2 }
}
```
**Ожидаемый результат:** 400 Bad Request.

---

### TC-006: Граничное значение sellerID (нижняя граница — 111111)
**Шаги:**
1. Отправить POST-запрос на `/api/1/item` с `sellerID: 111111`.
2. Тело запроса:
```json
{
  "sellerID": 111111,
  "name": "Товар с минимальным sellerID",
  "price": 500,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 1 }
}
```
**Ожидаемый результат:** 200 OK. Нижняя граница допустимого диапазона sellerID принимается.

---

### TC-007: Создание без обязательного поля statistics
**Шаги:**
1. Отправить POST-запрос на `/api/1/item`.
2. Тело запроса (объект `statistics` отсутствует):
```json
{
  "sellerID": 555666,
  "name": "Товар без статистики",
  "price": 100
}
```
**Ожидаемый результат:** 400 Bad Request.

---

### TC-008: Максимальная цена (граничный/стресс кейс)
**Шаги:**
1. Отправить POST-запрос на `/api/1/item` с `price: 999999999999999`.
2. Тело запроса:
```json
{
  "sellerID": 555666,
  "name": "Дорогой товар",
  "price": 999999999999999,
  "statistics": { "likes": 0, "viewCount": 0, "contacts": 0 }
}
```
**Ожидаемый результат:** 200 OK или 400 Bad Request. Сервер не должен падать; число должно быть корректно обработано или отклонено.

---

### TC-009: Спецсимволы в поле name (XSS-проверка)
**Шаги:**
1. Отправить POST-запрос на `/api/1/item`.
2. Тело запроса:
```json
{
  "sellerID": 555666,
  "name": "<script>alert(1)</script>",
  "price": 100,
  "statistics": { "likes": 0, "viewCount": 0, "contacts": 0 }
}
```
**Ожидаемый результат:** 200 OK (с экранированием) или 400 Bad Request. Сервер не должен возвращать скрипт в ответе без экранирования.

---

### TC-010: Очень длинное имя (1000+ символов)
**Шаги:**
1. Отправить POST-запрос на `/api/1/item` с `name` длиной 1010 символов.

**Ожидаемый результат:** 400 Bad Request. Сервер должен отклонять слишком длинные значения поля `name`.

---

## GET /api/1/item/{id}

### TC-101: Получение существующего объявления — проверка структуры ответа
**Шаги:**
1. Создать объявление через POST, сохранить UUID:
```json
{
  "sellerID": <случайный 111111–999999>,
  "name": "Ноутбук",
  "price": 80000,
  "statistics": { "likes": 3, "viewCount": 10, "contacts": 1 }
}
```
2. Отправить GET-запрос на `/api/1/item/{id}`.

**Ожидаемый результат:** 200 OK. Тело ответа содержит поля: `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`.

---

### TC-102: Получение несуществующего объявления
**Шаги:**
1. Отправить GET-запрос на `/api/1/item/e4a685e1-b27b-4954-9540-66726fc6e73d`.

**Ожидаемый результат:** 404 Not Found.

---

### TC-103: Получение объявления с невалидным ID (не UUID)
**Шаги:**
1. Отправить GET-запрос на `/api/1/item/abc`.

**Ожидаемый результат:** 400 Bad Request.

---

### TC-104: Получение удалённого объявления (E2E)
**Шаги:**
1. Создать объявление через POST, сохранить UUID:
```json
{
  "sellerID": <случайный 111111–999999>,
  "name": "Велосипед",
  "price": 15000,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 1 }
}
```
2. Удалить объявление через DELETE `/api/2/item/{id}`.
3. Отправить GET-запрос на `/api/1/item/{id}`.

**Ожидаемый результат:** 404 Not Found.

---

### TC-105: Соответствие типов данных в ответе (числа, не строки)
**Шаги:**
1. Создать объявление через POST, сохранить UUID:
```json
{
  "sellerID": <случайный 111111–999999>,
  "name": "Смартфон на Android",
  "price": 25000,
  "statistics": { "likes": 5, "viewCount": 20, "contacts": 2 }
}
```
2. Выполнить GET-запрос на `/api/1/item/{id}`.

**Ожидаемый результат:** 200 OK. Поля `price`, `sellerId`, `likes`, `viewCount`, `contacts` являются целыми числами (integer), а не строками.

---

## GET /api/1/{sellerID}/item

### TC-201: Успешное получение всех объявлений продавца
**Шаги:**
1. Создать два объявления с одним `sellerID`:
```json
{ "sellerID": <случайный 111111–999999>, "name": "Товар А", "price": 1000, "statistics": { "likes": 1, "viewCount": 1, "contacts": 1 } }
{ "sellerID": <тот же sellerID>, "name": "Товар Б", "price": 2000, "statistics": { "likes": 2, "viewCount": 2, "contacts": 2 } }
```
2. Отправить GET-запрос на `/api/1/{sellerID}/item`.

**Ожидаемый результат:** 200 OK. Тело ответа — массив объявлений. Все объявления принадлежат `{sellerID}`.

---

### TC-202: Несуществующий продавец — пустой массив
**Шаги:**
1. Отправить GET-запрос на `/api/1/9999989/item`.

**Ожидаемый результат:** 200 OK. Пустой массив `[]`.

---

### TC-203: Неверный формат sellerID (строка вместо числа)
**Шаги:**
1. Отправить GET-запрос на `/api/1/abc/item`.

**Ожидаемый результат:** 400 Bad Request.

---

## DELETE /api/2/item/{id}

### TC-401: Удаление существующего объявления
**Шаги:**
1. Создать объявление через POST, сохранить UUID:
```json
{
  "sellerID": <случайный 111111–999999>,
  "name": "Товар для удаления",
  "price": 3000,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 1 }
}
```
2. Отправить DELETE на `/api/2/item/{id}`.

**Ожидаемый результат:** 200 OK. Поле `status` подтверждает удаление.

---

### TC-402: Повторное удаление — идемпотентность DELETE
**Шаги:**
1. Выполнить DELETE для уже удалённого `{id}` из TC-401.

**Ожидаемый результат:** 404 Not Found (ресурс уже не существует).
