# Баг-репорты

## BUG-001: POST /api/1/item — объявление создаётся с пустым name

**Краткое описание:** Сервис принимает и сохраняет объявление с пустым полем `name`, вместо того чтобы вернуть ошибку валидации.

**Шаги воспроизведения:**
1. Отправить POST-запрос на `https://qa-internship.avito.com/api/1/item`.
2. Тело запроса:
```json
{
  "sellerID": 555123,
  "name": "",
  "price": 6000,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 2 }
}
```

**Фактический результат:** 200 OK. Объявление создаётся с пустым именем.

**Ожидаемый результат:** 400 Bad Request. Поле `name` обязательно и не должно быть пустым.

**Серьёзность:** Major

**Окружение:** `https://qa-internship.avito.com`, Go 1.24, Windows 11

## BUG-006: POST /api/1/item — API зависает при sellerID = 111111

**Краткое описание:** При отправке граничного значения `sellerID: 111111` API не отвечает и зависает на 30+ секунд.

**Шаги воспроизведения:**
1. Отправить POST-запрос на `https://qa-internship.avito.com/api/1/item`.
2. Тело запроса:
```json
{
  "sellerID": 111111,
  "name": "Товар с минимальным sellerID",
  "price": 500,
  "statistics": { "likes": 1, "viewCount": 1, "contacts": 1 }
}
```

**Фактический результат:** Запрос зависает, ответ не приходит (timeout).

**Ожидаемый результат:** 200 OK. Нижняя граница диапазона 111111–999999 должна быть допустима согласно условию задания.

**Серьёзность:** Critical

**Окружение:** `https://qa-internship.avito.com`, Go 1.24, Windows 11

## BUG-008: GET /api/1/statistic/{id} — статистика доступна после удаления объявления

**Краткое описание:** После удаления объявления через DELETE /api/2/item/{id} статистика по этому ID всё ещё доступна.

**Шаги воспроизведения:**
1. Создать объявление через POST /api/1/item, получить {id}.
2. Удалить объявление через DELETE /api/2/item/{id}.
3. Запросить статистику через GET /api/1/statistic/{id}.

**Фактический результат:** 200 OK. Статистика возвращается.

**Ожидаемый результат:** 404 Not Found. Статистика удалённого объявления не должна быть доступна.

**Серьёзность:** Major (утечка данных)

**Окружение:** `https://qa-internship.avito.com`, Go 1.24, Windows 11

---

## BUG-009: DELETE /api/2/item/{id} — ответ не содержит поле status

**Краткое описание:** При успешном удалении объявления API возвращает 200 OK, но тело ответа не содержит поле `status` с подтверждением.

**Шаги воспроизведения:**
1. Создать объявление через POST /api/1/item, получить {id}.
2. Отправить DELETE на /api/2/item/{id}.

**Фактический результат:** 200 OK. Тело ответа пустое или не содержит поле `status`.

**Ожидаемый результат:** 200 OK. Тело ответа содержит `{"status": "Объявление удалено"}` или аналогичное подтверждение.

**Серьёзность:** Minor (несоответствие документации)

**Окружение:** `https://qa-internship.avito.com`, Go 1.24, Windows 11
